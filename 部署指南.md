## 1. 硬件与软件要求

## 2. 环境准备

### 2.1 检查 Windows 版本

1. 按 `Win + R` 打开运行窗口
2. 输入 `winver` 并回车
3. 确认版本号 >= 19044（Windows 10 21H2 或更高）

### 2.2 启用 BIOS 虚拟化

**重要**：运行 Docker 需要 BIOS 中启用虚拟化技术。

1. 重启电脑，进入 BIOS 设置（通常按 F2、F10、F12 或 Delete 键）
2. 找到虚拟化选项（名称可能是：Intel VT-x、AMD-V、Virtualization Technology）
3. 设置为 **Enabled**
4. 保存并退出 BIOS

### 2.3 检查 CPU 虚拟化状态（Windows 中）

打开任务管理器 → 性能 → CPU，查看右下角"虚拟化"是否显示"已启用"。

## 3. 安装必要软件

### 3.1 安装 WSL2

wsl --update

## 3.2 安装 Docker Desktop

**步骤 1**：下载安装程序

访问 [Docker Desktop 4.49版本](https://docs.docker.com/desktop/release-notes/?utm_source=copilot.com#4490)，下载

**步骤 2**：运行安装程序

1. 双击下载的 `Docker Desktop Installer.exe`
2. 安装完成后点击 "Close and restart"

**步骤 3**：重启电脑后启动 Docker Desktop

1. 从开始菜单搜索 "Docker Desktop" 并启动
2. 接受服务协议
3. 跳过教程（可选）
4. 等待 Docker 引擎启动完成（任务栏 Docker 图标显示 "Engine running"）

**步骤 4**：配置 Docker Desktop（可选但推荐）

1. 右键任务栏 Docker 图标 → Settings
2. Resources → Advanced
3. 调整配置：
   - **Memory**: 建议 12GB+（最低 8GB）
   - **CPU**: 建议 6 核+（最低 4 核）
   - **Disk image location**: 建议放在 SSD 上
4. 点击 "Apply & Restart"

**步骤 5**：验证 Docker 安装

打开 cmd：

cmd
docker --version
# 期望输出: Docker version 24.x.x 或更高

docker compose version
# 期望输出: Docker Compose version v2.x.x

docker info
# 查看详细信息，无错误即可


---

### 3.3 安装 Node.js

**步骤 1**：下载安装程序

访问 <https://nodejs.org/> ，下载 **18.x LTS** 版本（Windows Installer 64-bit）

**步骤 2**：运行安装程序

1. 双击下载的 `.msi` 文件
2. 点击 "Next"
3. 勾选 "I accept the terms..."，点击 "Next"
4. 保持默认安装路径，点击 "Next"
5. **重要**：确保 **"Add to PATH"** 选项被勾选
6. 一路点击 "Next" 直到 "Install"
7. 完成后点击 "Finish"

**步骤 3**：验证安装

**关闭并重新打开 cmd**，然后执行：

cmd
node --version
# 期望输出: v18.x.x

npm --version
# 期望输出: 9.x.x 或 10.x.x


---

### 3.4 安装 Git for Windows

**步骤 1**：下载安装程序

访问 [Git for Windows](https://git-scm.com/download/win)，下载会自动开始

**步骤 2**：运行安装程序

1. 双击 `Git-2.x.x-64-bit.exe`
2. 点击 "Next"
3. 保持默认安装路径，点击 "Next"
4. 组件选择保持默认，点击 "Next"
5. 开始菜单文件夹保持默认，点击 "Next"
6. 编辑器选择保持默认（或选择你喜欢的），点击 "Next"
7. **PATH 环境**：选择 **"Git from the command line and also from 3rd-party software"**
8. 其余选项保持默认，一直点击 "Next" 直到 "Install"
9. 完成后点击 "Finish"

**步骤 3**：验证安装

打开 cmd：

cmd
git --version
# 期望输出: git version 2.x.x.windows.1

## 5. 启动系统

### 5.1 启动顺序

系统启动需要按以下顺序执行：

1. **Docker Desktop** → 2. **区块链网络** → 3. **后端服务** → 4. **前端服务**

### 5.2 启动 Docker Desktop

确保 Docker Desktop 已启动且引擎运行中（任务栏图标显示绿色）。

### 5.3 启动区块链网络

分步执行：

cmd
# 步骤 1: 启动容器
cd C:\Projects\code\blockchain\network\
docker compose up -d

# 等待约 30 秒让容器完全启动

# 步骤 2: 创建通道
cd C:\Projects\code\blockchain\network\scripts
.\createChannel.bat

# 步骤 3: 部署链码
.\deployChaincode.bat


### 5.3.1 注册Fabric用户（必须）

该项目使用Fabric钱包身份，需要在后端运行前完成：

cmd
cd C:\Projects\code\backend
npm run enroll

运行结束后，确认以下目录存在（特别是 `2.id`）：

- D:\medicalsupplychain\blockchain\wallet\producer\2.id
- D:\medicalsupplychain\blockchain\wallet\distributor\2.id
- D:\medicalsupplychain\blockchain\wallet\hospital\2.id
- D:\medicalsupplychain\blockchain\wallet\regulator\2.id

如果 `2.id` 缺失，说明CA未生成 `User2`，请对应组织使用CA注册并enroll后重跑该命令。


### 5.4 验证区块链网络

cmd
docker ps


应看到约 **21 个容器** 运行中，包括：

| 容器类型 | 数量 | 名称示例 |
| ---------- | ------ | ---------- |
| Orderer 节点 | 3 | orderer1.supplychain.com |
| Peer 节点 | 8 | peer0.producer.supplychain.com |
| CouchDB | 4 | couchdb0.producer |
| CLI 工具 | 1 | cli |

### 5.5 启动后端服务

**打开新的 cmd 窗口**（保持区块链网络运行）：

cmd
cd C:\Projects\code\backend
npm run dev


后端服务地址：[http://localhost:3000](http://localhost:3000)

### 5.6 启动前端服务

**再打开新的 PowerShell 窗口**：

powershell
cd C:\Projects\code\frontend
npm run dev


前端服务地址：[http://localhost:5173](http://localhost:5173)

## 7. 日常操作

### 7.1 每次启动系统

按顺序执行：

1. 启动 **Docker Desktop**（等待引擎运行）
2. 运行区块链启动脚本：
   cmd
   cd C:\Projects\code\blockchain\network
   docker compose up -d
   
3. 启动后端服务：
   cmd
   cd C:\Projects\code\backend
   npm run dev
   
4. 启动前端服务：

   cmd
   cd C:\Projects\code\frontend
   npm run dev
   

### 7.2 停止系统

**停止前后端服务**：
在运行服务的 cmd 窗口按 `Ctrl + C`

**停止区块链网络**：
powershell
cd C:\Projects\code\blockchain\network
docker compose down


### 7.3 完全清理（重置系统）

如果需要完全重置区块链网络：

powershell
cd C:\Projects\code\blockchain\network
docker compose down -v
docker volume prune -f


> **警告**：此操作会清除所有区块链数据！

### 7.4 查看日志

**查看区块链容器日志**：
powershell
# 查看所有容器状态
docker ps

# 查看特定容器日志
docker logs peer0.producer.supplychain.com
docker logs cli


**查看后端日志**：
后端日志直接输出在 cmd 窗口中。

## 8. 常见问题解决

### 问题 1：Docker Desktop 无法启动

**现象**：Docker Desktop 一直显示 "Starting..." 或报错

**解决方案**：

1. 确保 WSL2 已正确安装：

   powershell
   wsl --status
   

2. 重启 Docker Desktop

3. 如果仍然失败，重置 Docker：
   - 右键任务栏 Docker 图标
   - 选择 "Troubleshoot"
   - 点击 "Reset to factory defaults"
   - 重启电脑

4. 检查 Windows 服务：
   - 按 `Win + R`，输入 `services.msc`
   - 确保 "Docker Desktop Service" 正在运行

### 问题 2：WSL2 安装失败

**现象**：WSL2 启用命令报错

**解决方案**：

1. 确保 Windows 版本 >= 19044
2. 确保 BIOS 中启用了虚拟化（Intel VT-x / AMD-V）
3. 使用管理员权限运行 PowerShell
4. 手动安装 WSL2：

   powershell
   wsl --install
   

### 问题 3：npm install 失败或速度慢

**现象**：依赖安装报错或超时

**解决方案**：

powershell
# 设置国内镜像
npm config set registry https://registry.npmmirror.com

# 清理缓存
npm cache clean --force

# 删除 node_modules 重新安装
Remove-Item -Recurse -Force node_modules
Remove-Item package-lock.json
npm install


### 问题 4：端口被占用

**现象**：启动报错 "Port 3000 is already in use" 或 "Port 5173 is already in use"

**解决方案**：

powershell
# 查找占用 3000 端口的进程
netstat -ano | findstr :3000

# 结果示例：
# TCP    0.0.0.0:3000    0.0.0.0:0    LISTENING    12345

# 杀掉进程（12345 是进程ID）
taskkill /PID 12345 /F

# 同样方法处理 5173 端口
netstat -ano | findstr :5173
taskkill /PID <进程ID> /F


### 问题 5：区块链脚本执行失败

**现象**：createChannel.bat 或 deployChaincode.bat 报错

**解决方案**：

1. 确保所有 Docker 容器都在运行：

   powershell
   docker ps
   

2. 检查容器数量是否正确（应有约 21 个）

3. 查看容器日志：

   powershell
   docker logs cli
   

4. 尝试在 **Git Bash** 中运行脚本：
   - 打开 Git Bash
   - 执行：

     bash
     cd /c/Projects/code/blockchain/network
     ./scripts/createChannel.sh
     ./scripts/deployChaincode.sh
     

### 问题 6：后端连接区块链失败

**现象**：后端日志显示 "Failed to connect to fabric network"

**解决方案**：

1. 检查区块链网络状态：

   powershell
   docker ps | findstr peer
   

2. 确保链码已部署成功：

   powershell
   docker exec -i cli peer lifecycle chaincode querycommitted --channelID supplychain-channel
   

3. 重启区块链网络：

   powershell
   cd C:\Projects\code\blockchain\network\scripts
   docker compose down 
   docker compose up -d
   

### 问题 7：前端页面空白或报错

**现象**：访问 <http://localhost:5173> 显示空白页或有错误

**解决方案**：

1. 打开浏览器开发者工具（按 F12）
2. 查看 Console 是否有错误
3. 查看 Network 是否有请求失败
4. 确保后端服务已启动（<http://localhost:3000/health> 能访问）
5. 清除浏览器缓存后刷新

### 问题 8：Docker 内存不足

**现象**：容器启动失败或系统卡死

**解决方案**：

1. 打开 Docker Desktop → Settings → Resources
2. 增加 Memory 到 12GB 或更多
3. 点击 "Apply & Restart"

### 问题 9：登录失败

**现象**：输入正确账号密码但仍登录失败

**解决方案**：

1. 检查后端服务是否正常运行
2. 检查区块链网络是否正常
3. 查看后端控制台是否有错误日志
4. 尝试使用其他账号登录
