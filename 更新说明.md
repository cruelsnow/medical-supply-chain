# 更新说明（2026-04-15）

## 修复内容

### 1. 用户管理权限优化
- **监管方可跨组织管理**：监管方管理员可以创建、编辑、重置密码、删除其他组织的用户
- **禁止操作自身账号**：管理员不能编辑或删除自己的账号
- **组织隔离**：非监管方只能查看和操作本组织内的用户
- **前端权限控制**：侧边栏菜单按角色过滤，非管理员不可见用户管理入口

### 2. 资产登记增加数量字段
- **链码**：MedicalAsset 结构体增加 Quantity 字段，InitAsset 函数增加 quantity 参数及校验
- **后端**：验证器、服务层、Mock模式参数映射均支持 quantity

### 3. 订单详情查看
- **全部角色**：医院、生产商、经销商、监管方的订单管理页面均可点击"查看详情"
- **详情抽屉**：展示订单基本信息、货品明细表（名称/规格/数量/单价/UDI）、物流信息（运输单号）、状态时间线

### 4. 经销商页面合并
- **发货管理合并到订单管理**：删除独立的发货管理（ShipManage）和收货确认（Receive）页面
- **3 Tab 布局**：待处理订单 / 发货操作 / 仓储清单
- **后端支持**：发货时可关联 UDI 编号到订单项

### 5. 医院订单管理优化
- **入库合并到订单**：删除独立的验收入库页面，功能合并到采购订单详情中
- **一键入库**：订单状态为"已送达"时，详情中显示"确认收货入库"按钮，点击后自动完成资产入库
- **UDI 和运输单号展示**：订单详情中展示关联的 UDI 编号和运输单号

### 6. 哈希校验功能完善
- **生产商上传质检报告**：资产登记时可上传质检报告（PDF/图片），系统自动计算 SHA-256 哈希并上链
- **监管方文档校验**：输入 UDI → 查询链上哈希 → 上传原始文档 → 自动对比校验
- **二进制哈希计算**：使用 ArrayBuffer 读取文件，确保 PDF/图片等二进制文件的哈希计算准确

### 7. 其他修复
- **Mock模式参数映射**：修复 InitAsset 在 mock 模式下参数索引错位问题

## 升级部署说明（客户已有环境）

本交付包为完整最新代码，直接使用即可。由于客户环境已部署 v1.0 链码，需要升级链码：

### 步骤1：部署后端
```cmd
cd backend
npm install
npm run dev
```

### 步骤2：部署前端
```cmd
cd frontend
npm install
npm run dev
```

### 步骤3：升级链码（在客户电脑上执行）
```cmd
cd blockchain\network\scripts
upgradeChaincode.bat
```
该脚本会以 v1.1/sequence 2 升级链码
