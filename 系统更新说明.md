## 更新内容

### 1. 新增功能

### 2. 模拟数据模式

系统支持在**无区块链网络**的情况下运行，使用模拟数据进行测试。

#### 开启方式

**方式一：环境变量（推荐）**
```bash
# 在 backend 目录下创建 .env 文件
cd backend
echo "MOCK_MODE=true" > .env

# 然后启动服务
npm run dev
```

**方式二：修改配置文件**
```bash
# 编辑 backend/src/config/index.ts
# 将 mockMode.enabled 设为 true
```

**方式三：自动降级**
- 区块链网络不可用时，系统会**自动切换**到模拟模式
- 无需手动配置

#### 模拟模式特点

| 特性 | 说明 |
|------|------|
| 数据 | 使用预设的5条模拟资产数据 |
| 响应 | 100ms 延迟，模拟真实网络 |
| 操作 | 所有 API 正常响应，数据保存在内存中 |
| 重启 | 重启后数据重置 |

#### 关闭模拟模式

```bash
# 删除 .env 文件或设置为 false
echo "MOCK_MODE=false" > .env
```

### 3. 权限说明

#### 角色对比

| 功能 | viewer（查看者） | operator（操作员） | admin（管理员） |
|------|:----------------:|:------------------:|:---------------:|
| **查看仪表盘** | ✅ | ✅ | ✅ |
| **查看资产列表** | ✅ | ✅ | ✅ |
| **全链追溯** | ✅ | ✅ | ✅ |
| **哈希校验** | ✅ | ✅ | ✅ |
| **数据统计** | ✅ | ✅ | ✅ |
| **资产登记上链** | ❌ | ✅ | ✅ |
| **发货操作** | ❌ | ✅ | ✅ |
| **收货确认** | ❌ | ✅ | ✅ |
| **环境监控录入** | ❌ | ✅ | ✅ |
| **验收入库** | ❌ | ✅ | ✅ |
| **临床核销** | ❌ | ✅ | ✅ |
| **资产召回** | ❌ | ✅ | ✅ |
| **用户管理** | ❌ | ❌ | ✅ |
| **创建用户** | ❌ | ❌ | ✅ |
| **重置密码** | ❌ | ❌ | ✅ |

#### 权限等级

```
viewer (等级1) < operator (等级2) < admin (等级3)
```

- **viewer**：只读权限，适合审计、监管人员查看数据
- **operator**：业务操作权限，适合日常业务操作人员
- **admin**：完全权限，适合系统管理员

#### 页面表现

| 角色 | 操作按钮状态 | 权限提示 |
|------|-------------|---------|
| viewer | **禁用**（灰色不可点击） | 页面顶部显示"权限不足"提示 |
| operator | 正常可用 | 无提示 |
| admin | 正常可用 | 无提示 |

## 更新操作步骤

### 前提条件
系统已部署过区块链网络，本次**只需更新前后端代码**。

### 步骤 1：备份旧代码
```bash
# 备份现有代码（以防需要回滚）
cp -r backend backend.backup
cp -r frontend frontend.backup
```

### 步骤 2：解压新代码
```bash
unzip medical-supply-chain-code-20260328.zip
```

### 步骤 3：更新后端
```bash
cd backend

# 安装/更新依赖
npm install

# 重启后端服务
npm run dev

# 或者如果是生产环境
npm run build && npm start
```

### 步骤 4：更新前端
```bash
cd frontend

# 安装/更新依赖
npm install

# 重启前端服务
npm run dev

# 或者如果是生产环境
npm run build
```

cd backend && npm run dev
