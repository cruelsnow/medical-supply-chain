# 基于区块链的医用耗材供应链管理系统

## 项目概述

### 项目背景

医用耗材在医疗行业中扮演着至关重要的角色，其质量安全直接关系到患者的生命健康。然而，传统的医用耗材供应链管理存在以下问题：

1. **信息不透明**：供应链各环节信息孤岛，难以实现全程追溯
2. **数据易篡改**：传统中心化存储方式存在数据被篡改的风险
3. **监管困难**：监管部门难以实时掌握耗材流转状态
4. **信任缺失**：各参与方之间缺乏有效的信任机制

### 项目目标

本系统基于 **Hyperledger Fabric** 区块链技术，构建一个去中心化、防篡改、可追溯的医用耗材供应链管理平台，实现：

- 耗材全生命周期追踪
- 多方协作的信任机制
- 实时监管与预警
- 数据不可篡改与可审计

## 系统功能

### 1. 生产商功能

#### 1.1 资产登记
- **功能描述**：生产商对医用耗材进行赋码登记，生成唯一标识（UDI）
- **上链数据**：
  - UDI编码（唯一标识）
  - 耗材名称、规格、型号
  - 批次号、生产日期、有效期
  - 生产商信息
  - 资质证明哈希
- **业务价值**：实现耗材源头赋码，确保数据不可篡改

#### 1.2 发货管理
- **功能描述**：管理发往经销商的耗材发货记录
- **上链数据**：
  - 发货单号
  - 耗材UDI列表
  - 收货方信息
  - 发货时间、物流信息
- **业务价值**：记录权属转移，实现物流追踪

#### 1.3 资产列表
- **功能描述**：查看已登记的所有耗材资产
- **查询维度**：
  - 按状态筛选（已创建、在途、在库、已消耗、召回）
  - 按批次筛选
  - 按时间范围筛选

### 2. 经销商/物流功能

#### 2.1 收货确认
- **功能描述**：确认接收生产商发来的耗材
- **上链数据**：
  - 收货时间
  - 收货人信息
  - 验收结果
- **业务价值**：完成权属转移确认

#### 2.2 发货管理
- **功能描述**：管理发往医院的耗材发货记录

#### 2.3 环境监控
- **功能描述**：记录运输过程中的环境数据
- **上链数据**：
  - 温度、湿度
  - 位置信息
  - 异常标记
- **业务价值**：冷链监控，确保耗材质量

### 3. 医院功能

#### 3.1 验收入库
- **功能描述**：验收并入库耗材
- **上链数据**：
  - 入库时间
  - 验收结果
  - 存放位置

#### 3.2 库存管理
- **功能描述**：管理医院耗材库存
- **功能特性**：
  - 库存查询
  - 效期预警
  - 库存盘点

#### 3.3 临床核销
- **功能描述**：记录耗材在临床使用情况
- **上链数据**：
  - 使用科室
  - 手术/治疗编号
  - 操作人员
  - 使用时间
- **业务价值**：实现耗材最终核销，完成全生命周期追踪

### 4. 监管机构功能

#### 4.1 全链追溯
- **功能描述**：查询耗材从生产到使用的完整流转记录
- **追溯内容**：
  - 生产信息
  - 物流轨迹
  - 环境数据
  - 使用记录
- **业务价值**：一键追溯，快速定位问题

#### 4.2 哈希校验
- **功能描述**：校验资质文件是否被篡改
- **校验原理**：比对链上哈希与文件实际哈希
- **业务价值**：确保资质文件真实性

#### 4.3 数据统计
- **功能描述**：展示供应链运营数据
- **统计维度**：
  - 耗材总量统计
  - 各状态占比
  - 异常预警统计
  - 各组织交易量

### 组织架构

```
┌─────────────────────────────────────────────────────────────┐

│  ┌─────────────────────────────────────────────────────┐   │

│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐       │
│  │ Producer │ │Distributor│ │ Hospital │ │ Regulator│       │
│  │   组织   │ │   组织    │ │   组织   │ │   组织   │       │
│  │          │ │           │ │          │ │          │       │
│  │ peer0    │ │ peer0     │ │ peer0    │ │ peer0    │       │
│  │ peer1    │ │ peer1     │ │ peer1    │ │ peer1    │       │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘       │
└─────────────────────────────────────────────────────────────┘
```

### 网络配置

| 配置项 | 数值 | 说明 |
|--------|------|------|
| 排序节点 | 3 | Raft 共识，容错 1 节点 |
| 组织数量 | 4 | 生产商、经销商、医院、监管 |
| 每组织Peer数 | 2 | 主备节点 |
| 通道数量 | 1 | supplychain-channel |
| 链码数量 | 1 | supplychain |
| 状态数据库 | CouchDB | 支持富查询 |

### 共识机制

采用 **Raft** 共识算法：
- 基于 Leader-Follower 模式
- 支持 3 节点容错 1 节点故障
- 交易确认延迟低
- 适合联盟链场景

## 安全设计

### 1. 身份认证

- **PKI 证书体系**：每个组织使用独立的 CA 签发证书
- **MSP 成员服务**：基于 X.509 证书的身份验证
- **JWT Token**：后端 API 认证机制

### 2. 数据安全

- **TLS 加密**：节点间通信加密
- **私钥保护**：私钥存储在本地钱包
- **数据不可篡改**：区块链哈希链式结构

### 3. 访问控制

- **基于角色的权限**：不同角色访问不同功能
- **组织隔离**：各组织只能操作自己的数据
- **链码级别控制**：背书策略控制交易有效性

## 项目亮点

### 1. 技术创新

- **联盟链架构**：采用 Hyperledger Fabric 联盟链，兼顾去中心化与性能
- **Raft 共识**：使用 Raft 共识算法，交易确认延迟低
- **富查询支持**：使用 CouchDB 状态数据库，支持复杂查询
- **多组织协作**：4 个组织、8 个 Peer 节点的真实联盟场景

### 2. 业务价值

- **全程追溯**：从生产到使用的完整生命周期追踪
- **防篡改**：区块链确保数据不可篡改
- **信任机制**：多方协作的信任基础
- **监管透明**：监管部门实时掌握供应链状态

### 3. 工程实践

- **容器化部署**：全部服务 Docker 容器化
- **微服务架构**：前后端分离，模块化设计
- **类型安全**：TypeScript 全栈开发
- **代码规范**：统一的代码风格和注释规范

本项目成功实现了一个基于 Hyperledger Fabric 的医用耗材供应链管理系统，解决了传统供应链管理中的信息不透明、数据易篡改、监管困难等问题。系统采用联盟链架构，支持生产商、经销商、医院、监管机构四方协作，实现了医用耗材从生产登记到临床使用的全生命周期追溯，为医疗行业的数字化转型提供了可靠的区块链解决方案。
