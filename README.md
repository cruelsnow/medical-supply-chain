# 系统更新说明（2026-04-12）

## 本次更新概述

本次更新针对反馈的 5 个问题进行修复和优化。

---

## 一、修改内容详细说明

### 修改1：订单详情查看（各角色）

**问题**：所有角色（生产商/经销商/医院/监管）的订单管理页面，无法查看订单详情。

**修复方案**：
- 生产商、经销商、医院、监管的订单管理页面均增加了「订单详情抽屉」
- 点击订单列表中的操作按钮，右侧弹出抽屉展示完整订单信息

**详情内容包含**：
- 基本信息：订单编号、标题、状态、金额、创建时间
- 货品明细表：名称、规格、数量、单价、UDI编号
- 物流信息：运输单号、发货时间、送达时间
- 订单时间线：状态变更记录

**涉及文件**：
| 文件 | 改动 |
|------|------|
| `frontend/src/views/producer/OrderManage.vue` | 增加订单详情抽屉 |
| `frontend/src/views/regulator/OrderAudit.vue` | 增加订单详情抽屉（含时间线） |

---

### 修改2：经销商发货与订单管理合并

**问题**：经销商的「发货管理」和「订单管理」是两套独立系统，数据无法对应。发货时需要输入 UDI 编号和运输单号。需要增加确认收货后的仓储清单。

**修复方案**：
- 将经销商发货管理、收货确认功能全部合并到订单管理页面
- 订单管理页面改为 3 个 Tab 布局
- 删除旧的发货管理和收货确认页面

**3 Tab 布局说明**：
| Tab | 功能 |
|-----|------|
| 待处理订单 | 查看/确认/拒绝订单，查看订单详情 |
| 发货操作 | 对已确认订单进行发货，输入运输单号和每项货品的 UDI 编号 |
| 仓储清单 | 查看已送达入库的货品清单 |

**涉及文件**：
| 文件 | 改动 |
|------|------|
| `frontend/src/views/distributor/OrderManage.vue` | 完整重写，3 Tab 布局 |
| `frontend/src/router/index.ts` | 删除旧路由（distributor/receive, distributor/ship） |
| `frontend/src/views/common/Layout.vue` | 删除侧边栏旧菜单项 |
| `frontend/src/controllers/order.controller.ts` | 发货接口支持 dispatchItems（UDI） |
| `frontend/src/services/order.service.ts` | 保存 UDI 到订单项，更新发货状态 |
| `frontend/src/api/index.ts` | 前端发货接口参数更新 |

---

### 修改3：用户管理权限修复

**问题**：用户管理模块测试时所有角色都能编辑，没有限制为仅管理员。不同组织的 admin 能看到其他组织的用户。可以禁用/删除自己的账号。

**修复方案**：

#### 3.1 权限限制（前后端双重保护）
- 后端：所有用户管理接口限制 `role === 'admin'`
- 前端：非 admin 角色隐藏操作按钮，显示「仅管理员可操作」标签

#### 3.2 组织数据隔离
- 用户列表仅展示当前用户所在组织的用户
- 禁用/编辑/重置密码/删除操作限制在本组织范围内
- 禁止操作其他组织的用户

#### 3.3 自操作保护
- 不能禁用自己的账号（前端按钮禁用 + 后端拒绝）
- 不能删除自己的账号
- 不能修改自己的角色
- 不能重置自己的密码

**涉及文件**：
| 文件 | 改动 |
|------|------|
| `backend/src/controllers/auth.controller.ts` | 组织隔离、自操作保护 |
| `frontend/src/views/admin/UserManage.vue` | 操作按钮权限控制、自操作禁用 |
| `frontend/src/views/common/Layout.vue` | 用户管理菜单移入各组织子菜单 |
| `frontend/src/composables/usePermission.ts` | 清理已删除路由的权限映射 |

---

### 修改4：医院采购订单与验收入库合并

**问题**：医院的订单数据需要跟经销商同步。验收入库功能应合并到采购订单中。医院端在订单详情页面需要看到 UDI 编号、运输单号等信息，并且能够确认收货入库。

**修复方案**：
- 删除独立的验收入库页面，功能合并到采购订单管理
- 订单详情展示 UDI 编号、运输单号、物流状态
- 订单状态为「已送达」时，显示「确认收货入库」按钮
- 确认后自动完成资产入库，可在库存管理中查看

**涉及文件**：
| 文件 | 改动 |
|------|------|
| `frontend/src/views/hospital/OrderList.vue` | 完整重写，合并入库功能 |
| `frontend/src/router/index.ts` | 删除 hospital/inbound 路由 |
| `frontend/src/views/common/Layout.vue` | 删除验收入库菜单项 |

---

### 修改5：哈希校验功能完善

**问题**：监管方哈希校验功能无法使用，前面没办法生成文档哈希，也没有文档上传入口。

**修复方案**：
- 哈希校验页面改为分步交互流程：

| 步骤 | 操作 |
|------|------|
| 第1步 | 输入 UDI 编号，查询链上存储的文档哈希 |
| 第2步 | 上传原始文档，系统自动计算 SHA-256 哈希 |
| 第3步 | 对比链上哈希与文档哈希，显示校验结果 |

- 校验成功：显示绿色通过标识
- 校验失败：显示红色警告，提示数据可能被篡改

**涉及文件**：
| 文件 | 改动 |
|------|------|
| `frontend/src/views/regulator/Verify.vue` | 完整重写，分步交互流程 |

## 六、测试验证建议

### 1. 验证订单详情

```
用 hospital_admin 登录 → 采购订单 → 点击任意订单的「查看详情」→ 右侧弹出详情抽屉
用 producer_admin 登录 → 订单管理 → 点击「查看详情」→ 确认信息完整
用 regulator_admin 登录 → 订单审计 → 点击「查看详情」→ 确认含时间线
```

### 2. 验证经销商订单管理

```
用 distributor_admin 登录 → 物流管理 → 订单管理
  - Tab1「待处理订单」：查看订单列表，确认/拒绝订单
  - Tab2「发货操作」：选择已确认订单，输入运输单号和 UDI，发货
  - Tab3「仓储清单」：查看已入库的货品列表
```

### 3. 验证用户管理权限

```
用 distributor_admin 登录 → 物流管理 → 用户管理
  - 能看到自己组织的用户（distributor_op1、distributor_admin）
  - 自己账号的「编辑/禁用/删除/重置密码」按钮为灰色不可点击
  - 可以对其他用户进行编辑、禁用、重置密码等操作

用 distributor_op1 登录 → 确认看不到用户管理菜单
```

### 4. 验证医院采购订单

```
用 hospital_admin 登录 → 医院管理 → 采购订单
  - 查看订单列表
  - 点击「查看详情」→ 看到 UDI 编号、运输单号等物流信息
  - 如果订单状态为「已送达」→ 显示「确认收货入库」按钮
  - 确认入库后 → 去库存管理查看
```

### 5. 验证哈希校验

```
用 regulator_admin 登录 → 监管追溯 → 哈希校验
  - 第1步：输入 UDI 编码（如 UDI20240328001）→ 点击查询
  - 第2步：上传任意文档 → 系统自动计算哈希
  - 第3步：查看校验结果
  - 注：模拟数据中的哈希是预设的，上传不同文件会显示校验失败（正常）
```

## 四、与上一版本的主要区别

| 对比项 | 上一版本（04-10） | 本版本（04-12） |
|--------|------------------|----------------|
| 订单详情 | 无法查看 | 各角色均可查看详情抽屉 |
| 经销商管理 | 发货/订单分离 | 合并为 3 Tab 订单管理 |
| 用户管理 | 所有角色可编辑 | 仅 admin 可操作 + 组织隔离 |
| 医院入库 | 独立验收入库页面 | 合并到采购订单，详情展示 UDI |
| 哈希校验 | 无法使用 | 分步交互，查询-上传-对比 |
| 自操作保护 | 无 | 不能禁用/删除/修改自己 |
| 仪表盘按钮 | 指向已删除页面 | 指向正确的订单管理页面 |
